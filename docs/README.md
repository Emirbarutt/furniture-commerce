# Documentation

Durable project knowledge. Update the relevant document with any architectural, API, operational, or security change.

- `architecture/` — system design, module boundaries, and diagrams.
- `adr/` — immutable Architecture Decision Records.
- `api/` — API standards and published contracts.
- `runbooks/` — operational and incident-response procedures.
- `security/` — threat models, controls, and review guidance.
